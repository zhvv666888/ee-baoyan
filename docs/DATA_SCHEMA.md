# 项目数据字段规范

每一行必须表示同一年度、同一学院或研究所、同一培养项目的一份画像。不要把整所大学合并成一行。

## 核心字段

| 字段 | 含义 | 规则 |
|---|---|---|
| `program_id` | 稳定唯一标识 | 建议为学校缩写-学院-专业-年份 |
| `school` | 学校名称 | 使用官方全称 |
| `college` | 学院/研究所 | 使用当年官方名称 |
| `program_name` | 专业或项目 | 不混合不同专业代码 |
| `region` | 地区 | 省份或城市，保持统一 |
| `directions` | 研究方向 | 用 `|` 分隔 |
| `degree_types` | 培养类型 | academic_master / professional_master / direct_phd |
| `data_year` | 数据年份 | 画像对应年份 |
| `source_url` | 来源 | 优先官方公告 |
| `is_demo` | 是否演示 | 真实数据必须为 false |

## 画像字段

这些字段不是“官方录取线”，而是根据官方条件与匿名案例构造的可解释画像。任何数值都必须能回溯来源和计算过程。

- `min_rank_percent`：观察到或规则给出的基础排名线；
- `preferred_rank_percent`：更有竞争力的排名区间；
- `expected_school_tier`：0–5 的院校背景画像；
- `research_expectation`：0–5 的科研偏好；
- `competition_expectation`：0–5 的竞赛偏好；
- `english_min`：可验证的英语门槛；官方未给出时保持为空（`null`），不得填 0 伪装成“无门槛”；
- `required_strength`：0–100 的项目综合难度参数，仅供模型排序。

官方公告未明确的画像字段可以在 `published_programs` 中保持为空，并写入 `missing_fields_json`。推荐器会降低置信度并按高风险处理，明确提示“不能视为低门槛”，不会把空值转换为低要求。

## 证据等级

- **A**：学校/学院官方通知、公示、正式名单，且字段可直接验证；
- **B**：多个官方来源交叉验证，或有较多授权匿名案例；
- **C**：单一匿名案例、经验帖或不完整公开材料；
- **D**：未经验证的社区信息，仅能作为待核验线索。

不能找到的数据应保留为空或降级，不得猜测。

## 匿名案例

`cases.template.csv` 用于未来校准。严禁包含姓名、学号、联系方式、证件号码、精确宿舍地址等信息。
