# 开发路线图

## v0.1：流程验证（当前）

- 规则推荐；
- 演示数据；
- Web 表单与 API；
- 数据规范、校验和测试。

验收：任何贡献者可在本地启动并看到解释型冲稳保结果。

## v0.2：真实官方数据

- 覆盖 30–50 个电子信息强校的院系级项目；
- 按年份和申请阶段拆分；
- 所有记录带官方来源和访问日期；
- 增加人工审核状态和数据变更日志。

验收：系统可以只使用 A/B 级证据生成项目画像。

## v0.3：匿名案例

- 投稿表单；
- 隐私提示、授权和自动脱敏；
- 案例检索；
- 按项目显示背景分布而非公开个人明细。

验收：至少 300 条授权案例，并覆盖不同本科层次。

## v0.4：模型校准

- 相似案例基线；
- 逻辑回归或梯度提升模型；
- 按年份切分验证；
- 概率校准、漂移检测和偏差报告。

验收：公开模型卡，说明误差、缺失和不适用场景。

## v1.0：申请闭环

- 推荐组合；
- 截止日期；
- 院校申请看板；
- 导师与方向管理；
- 材料清单；
- 结果回传并参与下一年度校准。
