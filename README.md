# EE 保研罗盘（EE Baoyan Compass）

面向电子信息工程、通信工程及相近专业学生的开源保研项目匹配工具。

输入本科背景、排名、英语、科研竞赛和方向偏好，系统输出院系级别的 **冲刺 / 稳妥 / 保底** 建议，并展示匹配理由、风险项、数据证据等级和置信度。

> 当前版本是可运行 MVP。仓库自带的是**虚构演示院校画像**，只用于验证交互和推荐流程，绝不能直接用于真实保研决策。

## 为什么做这个项目

公开保研项目通常解决以下某一部分问题：

- 夏令营、预推免通知聚合；
- 院校、导师、材料和申请进度管理；
- 保研经验与面试资料整理；
- 面向考研的历史招生数据分析。

EE 保研罗盘希望补齐中间缺少的一层：

> 申请者背景 → 院校/学院/项目画像 → 可解释匹配 → 冲稳保组合。

推荐单位不是笼统的“学校”，而是：

`学校 + 学院/研究所 + 专业/项目 + 方向 + 年份 + 申请阶段`

## MVP 已实现

- 浏览器填写申请者背景；
- FastAPI 推荐接口；
- CSV 院校项目数据源；
- 基于规则的申请者强度与项目匹配评分；
- 冲刺、稳妥、保底分类；
- 匹配理由和主要风险解释；
- A/B/C/D 证据等级与置信度；
- 演示数据强制警告；
- 数据校验脚本；
- 推荐逻辑单元测试。

## 本地运行

### 1. 创建环境

```bash
python -m venv .venv
```

Windows：

```powershell
.venv\Scripts\activate
```

macOS / Linux：

```bash
source .venv/bin/activate
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动

```bash
uvicorn app.main:app --reload --port 8000
```

浏览器打开：`http://127.0.0.1:8000`

API 文档：`http://127.0.0.1:8000/docs`

也可以运行：

- Windows：双击 `start.bat`
- macOS / Linux：`bash start.sh`

## v0.2 官方网站采集

仓库已加入可配置的官方站点爬虫：

- 只允许配置中的学校官方域名；
- 遵守 robots.txt、限速、重试和缓存；
- 自动发现夏令营、推免、面试、优营和拟录取公告；
- 抽取截止时间、年级、英语、排名、名额和培养类型等明确字段；
- SQLite 增量去重，页面变化后重新进入审核队列；
- 名单类公告自动标记隐私敏感，不把姓名写入推荐数据；
- 采集结果必须人工审核后才能进入项目画像。

运行示例：

```bash
python scripts/crawl_official.py --list-sources
python scripts/crawl_official.py --source uestc-sice --since 2023-01-01
```

完整说明见 `docs/CRAWLER.md`。当前内置电子科技大学、北京邮电大学、东南大学和上海交通大学的首批来源配置；其余首批院校接入队列见 `config/source_backlog.csv`。

## 数据替换

1. 复制 `data/programs.template.csv`；
2. 按 `docs/DATA_SCHEMA.md` 填写真实项目；
3. 每一个门槛、画像或难度字段都应保留来源；
4. 将 `is_demo` 设置为 `false`；
5. 运行校验：

```bash
python scripts/validate_programs.py data/programs.csv
```

随后把 `app/repository.py` 中的默认数据路径切换到真实文件，或在后续版本中通过环境变量配置。

## 评分原则

项目刻意不输出录取概率。当前规则由四部分构成：

1. 申请者综合强度：院校层次、排名、GPA、英语、科研、论文、竞赛和项目；
2. 项目匹配度：方向、地区、培养类型和各项画像差距；
3. 风险档位：申请者强度与项目要求之间的裕量；
4. 证据置信度：来源级别、样本量、数据年份和是否为演示数据。

详细说明见 `docs/METHODOLOGY.md`。

## 数据伦理

- 不收集姓名、学号、身份证号、手机号等直接身份信息；
- 匿名案例必须获得贡献者授权；
- 不根据性别、民族、家庭背景等敏感属性推荐；
- 不抓取或公开违反网站规则、版权或个人隐私的数据；
- 不把论坛传言包装成官方门槛；
- 招生政策以学校和学院官方通知为准。

## 推荐的真实数据路线

### 阶段一：官方项目画像

收集近三年的：

- 夏令营和预推免通知；
- 报名资格与硬性条件；
- 入营、优营、复试和拟录取公示；
- 招生方向、培养类型和考核方式。

### 阶段二：匿名申请案例

建立公开表单，收集：

- 匿名本科背景；
- 申请项目与阶段；
- 入营、优营、录取结果；
- 是否套磁、面试形式和经验备注。

### 阶段三：校准模型

样本量足够后再引入：

- 基于相似申请者的案例检索；
- 可解释的逻辑回归或梯度提升模型；
- 时间衰减和年度漂移检测；
- 分方向、分学院的校准评估。

## 公告到推荐的数据发布闭环（v0.2）

本版本把官方公告处理为一条可审计、带人工闸门的流水线：

```text
官网公告 → notices/raw notice → program_drafts（待审核）
→ 本地人工审核 → published_programs → /api/recommend
```

爬虫会保存公告及结构化提取结果，并为非敏感公告创建 `pending` draft；缺失字段保持为空，不猜测。面试名单、优秀营员名单、拟录取名单等敏感公告只写入跳过原因，不生成项目 draft。只有 `reviewed` draft 可以发布，推荐器在 `APP_DATA_MODE=published` 下只读 `published_programs`，不会读取 raw、extracted、pending 或 rejected 数据。

审核员可以发布仍有未知画像字段的真实项目，但 `missing_fields` 会原样保留；推荐器会降低置信度并按高风险处理，明确提示“不能视为低门槛”，不会把未知字段转换成 0 或宽松条件。

启动前复制环境模板，并设置正式爬取身份：

```powershell
Copy-Item .env.example .env
# 编辑 .env，替换 CRAWLER_USER_AGENT 中的仓库地址和联系邮箱
```

应用数据模式：

- `APP_DATA_MODE=demo`：只读取 `data/programs.demo.csv`，用于产品演示。
- `APP_DATA_MODE=published`：只读取 `APP_PUBLISHED_DATABASE` 指向的 SQLite `published_programs`；没有已发布数据时返回空结果，并提示“当前没有已审核发布的真实院校项目数据。”。
- 其他值会导致应用启动失败。

审核入口是 `http://127.0.0.1:8000/admin/review`。这是无认证的本地管理页面，只能在本机使用，不得公开暴露。页面支持筛选、详情、字段编辑、审核通过、驳回、恢复、发布、下架和审核历史查看。

推荐分数仍然是可解释的排序分数，不是录取概率；正式数据必须能回溯到公告 URL、标题、日期、内容哈希、证据等级和审核记录。
